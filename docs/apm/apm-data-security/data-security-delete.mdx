If you accidentally ingest sensitive data, follow these steps to remove or redact the offending data:

1. Stop collecting the sensitive data.
    Use the **remedy** column of the <DocLink id="serverlessObservabilityApmDataSecurity" section="sensitive-fields">sensitive fields</DocLink> table to determine how to stop collecting
    the offending data.

1. Delete or redact the ingested data. With data collection fixed, you can now delete or redact the offending data:

* <DocLink id="serverlessObservabilityApmDataSecurity" section="redact-specific-fields">Redact specific fields</DocLink>
* <DocLink id="serverlessObservabilityApmDataSecurity" section="delete-((es))-documents">Delete ((es)) documents</DocLink>

<div id="redact-field-data"></div>

### Redact specific fields

To redact sensitive data in a specific field, use the [update by query API](((ref))/docs-update-by-query.html).

For example, the following query removes the `client.ip` address
from APM documents in the `logs-apm.error-default` data stream:

```console
POST /logs-apm.error-default/_update_by_query
{
  "query": {
    "exists": {
      "field": "client.ip"
    }
  }
  "script": {
    "source": "ctx._source.client.ip = params.redacted",
    "params": {
      "redacted": "[redacted]"
    }
  }
}
```

Or, perhaps you only want to redact IP addresses from European users:

```console
POST /logs-apm.error-default/_update_by_query
{
  "query": {
    "term": {
      "client.geo.continent_name": {
        "value": "Europe"
      }
    }
  },
  "script": {
    "source": "ctx._source.client.ip = params.redacted",
    "params": {
      "redacted": "[redacted]"
    }
  }
}
```

See [update by query API](((ref))/docs-update-by-query.html) for more information and examples.

### Delete ((es)) documents

<DocCallOut title="Warning" color="warning">
This will permanently delete your data.
You should test your queries with the [search API](((ref))/search-search.html) prior to deleting data.
</DocCallOut>

To delete an ((es)) document,
you can use the [delete by query API](((ref))/docs-delete-by-query.html).

For example, to delete all documents in the `apm-traces-*` data stream with a `user.email` value, run the following query:

```console
POST /apm-traces-*/_delete_by_query
{
  "query": {
    "exists": {
      "field": "user.email"
    }
  }
}
```

See [delete by query API](((ref))/docs-delete-by-query.html) for more information and examples.
